VINS-Fusion系统运行环境要求

  • 虚拟机平台VMware
  • Ubantu 16.04系统配置
  • ROS机器操作平台配置
  • 关键库配置

一、VMware+Ubantu16.04环境搭建

  • 下载VMware虚拟机软件

  • Ubuntu系统镜像文件下载(ubuntu-16.04.2-desktop 32位)
    下面提供几处可用镜像地址

    　中科大源
    http://mirrors.ustc.edu.cn/ubuntu-releases/16.04/
    　阿里云开源镜像站
    http://mirrors.aliyun.com/ubuntu-releases/16.04/

  • 下载好虚拟机之后开始新建Ubantu虚拟系统

    选择自定义安装
选择兼容性
根据项目的需要和PC的配置我选择1核处理器和2G内存，磁盘存储空间选择40G
系统镜像选择稍后安装操作系统Linux
给虚拟机命名并选择其存储位置
其他系统向导选择推荐即可

  • 安装Ubantu系统

     点击虚拟机设置，选择使用ISO镜像文件
 选择下载好的镜像文件地址
 点击开启此虚拟机
 进入界面后选择中文简体开始安装
 由于磁盘是新建的，直接清理整个磁盘并载入Ubantu
 进入界面默认地址在shanghai，选择语言完成安装

二、Ubantu切换国内镜像源

  • 首先备份初试默认源

       sudo cp /etc/apt/sources.list{,.bak}

  • 切换权限

       sudo su - root

  • 打开VIM编辑源文件

        vim /etc/apt/sources.list

   如果找不到命令则说明Ubantu未安装VIM
   则需要安装VIM工具（默认的gedit有权限问题不方便修改文本文件）

    sudo apt-get update
    sudo apt-get install vim-gtk

    打开源文件后将原有地址代码删除
    替换成新的源（下面提供了几个常用的国内源）：
    1、阿里源

    deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
    deb http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
    deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
    deb http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
     deb http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse
      deb-src http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
     deb-src http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
     deb-src http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
      deb-src http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
      deb-src http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse

  • 更新软件列表信息

       sudo apt-get update

  • 更新软件

       sudo apt-get upgrade

三、ROS操作平台的搭建

  • 设置部分

       打开Ubantu中系统设置，选择软件与更新
   全部勾选，码源地址选择清华镜像站

  • 安装源

      由于ROS官网的更新变动这个源地址最好不要使用默认的ROS地址
  在命令行中输入
    sudo sh -c '. /etc/lsb-release && echo "deb  
    http://mirrors.ustc.edu.cn/ros/ubuntu/ $DISTRIB_CODENAME main" > /etc/apt/sources.list.d/ros-latest.list'

  • 设置端口

        获取公钥
   sudo apt-key adv --keyserver 'hkp://keyserver.ubuntu.com:80' --recv-key C1CF6E31E6BADE8868B172B4F42ED6FBAB17C654

  • 更新软件列表

       sudo apt-get update

  • 开始安装

       sudo apt-get install ros-kinetic-desktop-full
   安装时间得看网络速度，我大概花了1个半小时，中途可能会由于下载速度过慢中断安装，命令上移重新载入即可继续下载

  • 安装依赖项

       sudo rosdep init
   rosdep update

    如果出现如下错误则是源地址被屏蔽了，则需在hosts文件中修改网络管理

ERROR: cannot download default sources list from:
https://raw.githubusercontent.com/ros/rosdistro/master/rosdep/sources.list.d/20-default.list
Website may be down.

则在命令行中输入

sudo vi /etc/hosts

并打开编辑模式在文件最末尾处加一句代码

199.232.28.133 raw.githubusercontent.com

  • 环境设置到默认路径

      echo "source /opt/ros/kinetic/setup.bash" >> ~/.bashrc
  source ~/.bashrc

  • 安装Rosinstall

       sudo apt install python-rosinstall python-rosinstall-generator python-wstool build-essential

四、ROS平台测试

到这里如果没有出现错误的话ROS操作平台应该差不多搭建完基础部分
可以用例程测试一下开发环境：

  • 加载环境设置文件

       source /opt/ros/kinetic/setup.bash

  • 创建并初始化文件目录

    需要工具catkin库来构建系统，因此先创建初始化catkin目录

       mkdir -p ~/catkin_ws/src

   cd ~/catkin_ws/src

   catkin_init_workspace

    进入所创建的工作目录，使用catkin_make命令来构建

       cd ~/catkin_ws/

   catkin_make

  • 加载与catkin构建有关的环境文件

       source ~/catkin_ws/devel/setup.bash

  • ROS的配置

    1、用gedit打开bashrc文件

       gedit ~/.bashrc

    2、打开后移动到最底部添加以下内容（xxx.xxx.xxx.xxx是用户的IP地址，会根据所连网络改变，使用ifconfig查看）。输入后，保存更改并退出gedit。

        # Set ROS Kinetic
   source /opt/ros/kinetic/setup.bash
   source ~/catkin_ws/devel/setup.bash
   # Set ROS Network
   export ROS_HOSTNAME=xxx.xxx.xxx.xxx
   export ROS_MASTER_URI=http://${ROS_HOSTNAME}:11311
   # Set ROS alias command
   alias cw='cd ~/catkin_ws'
   alias cs='cd ~/catkin_ws/src'
   alias cm='cd ~/catkin_ws && catkin_make'

    3、应用更改

       source ~/.bashrc

  • 测试ROS
    打开第一个终端

       roscore

    打开第二个终端

       rosrun turtlesim turtlesim_node
   出现小乌龟窗口

    打开第三个终端使用键盘控制小乌龟

       rosrun turtlesim turtle_teleop_key

    这时如果你的小乌龟会跟随你键盘输入的操作爬动，则说明ROS操作系统安装完好啦啦！！

五、小结

走了超多的坑，俺写一个博客记录一下正确的搭建顺序和出现过的错误，总结了一下比较精炼的安装命令，希望读者看了能有帮助，不用再踩一遍，作者水平有限，有错误的地方还请指点一下哦，下一篇博客中我会利用ROS操作系统尝试跑通港科大的VINS-Fusion项目（目前正在调试中），感谢耐心浏览♥
